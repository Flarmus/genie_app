# Genie App для генерации скрипта Engee

Это GUI-приложение на Julia (Genie), которое:

- принимает промпт пользователя;
- отправляет его в `POST /script/generate`;
- показывает полученный `script` в интерфейсе.

## Структура

- `app.jl` — сервер Genie и API-роут `/script/generate`.
- `public/index.html` — веб-интерфейс (поле промпта + вывод скрипта).
- `Project.toml` — зависимости Julia.

## Требования

- Julia 1.12+
- Установленные пакеты из `Project.toml`
- (Опционально) запущенный backend API, который принимает:

```json
{"prompt":"..."}
```

и возвращает:

```json
{"script":"..."}
```

## Установка и запуск

```powershell
cd C:\codex\engee_gui
julia --project=. -e "using Pkg; Pkg.instantiate()"
julia --project=. app.jl
```

После запуска открой в браузере:

- `http://127.0.0.1:8000`

## Запуск в среде Engee

1. Установить папку репозитория на компьютер и разархивировать.
2. Папку `genie_app-main` перенести в любое место файловой системы среды Engee и сохранить путь к перенесённой папке.
   *Проверить путь можно в файловом менеджере Engee, вверху (`вкладки "Файлы"`) он отражён. 
   Если папка перенесена в корень, путь будет выглядеть следующим образом: `/user/genie_app-main`.*
3. Установить в консоли переменную окружения для отправки запросов на backend api командой:
   ```julia
   ENV["SCRIPT_API_URL"]="YOUR_API_URL"
   ```
4. Запустить приложение командой в консоли:
   ```julia
   engee.genie.start("/user/genie_app-main", open_url=true)
   ```
   При параметре `open_url=true` Engee попытается открыть страницу.
   Также в консоли появится ссылка вида `https://engee.com/prod/user/<workspace>-<user>/genie/genie_app-main/`, по ней также можно открыть приложение.
5. Для остановки приложения необходимо в консоли выполнить:
   ```julia
   engee.genie.stop("/user/genie_app-main")
   ```


## Подключение к backend API

По умолчанию приложение работает в демо-режиме (если `SCRIPT_API_URL` не задан).

Чтобы отправлять запросы в реальный backend, перед запуском укажи URL:

```powershell
$env:SCRIPT_API_URL = "http://127.0.0.1:8001/script/generate"
julia --project=. app.jl
```

## Как протестировать

1. Открой `http://127.0.0.1:8000`.
2. Введи промпт в поле `Prompt`.
3. Нажми `Generate`.
4. Проверь, что в блоке `Script` появился ответ.

## Типовые проблемы

- В ответе `DEMO MODE: no SCRIPT_API_URL set`:
  - не задана переменная `SCRIPT_API_URL`.

- Ошибка `HTTP 500` в GUI:
  - backend запущен, но не может сгенерировать ответ (проверь его логи).

- Страница не открывается на `8000`:
  - порт занят другим процессом или Genie не запущен.

## Полезные переменные окружения

- `SCRIPT_API_URL` — URL backend-эндпоинта генерации.
- `HOST` — хост Genie (по умолчанию `127.0.0.1`).
- `PORT` — порт Genie (по умолчанию `8000`).
